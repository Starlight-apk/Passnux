# 项目架构概览

## 整体架构

Passnux 采用前后端分离架构：

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  前端       │────▶│  后端 API    │────▶│  AI 模型    │
│ (Web UI)    │◀────│ (服务层)     │◀────│ (API 网关)  │
└─────────────┘     └──────────────┘     └─────────────┘
                           │
                    ┌──────┴──────┐
                    │  数据库     │
                    └─────────────┘
```

## 核心组件

| 组件 | 职责 | 技术选型 |
| :--- | :--- | :--- |
| 前端 | 用户界面交互 | 待定 |
| 后端 | 业务逻辑处理 | 待定 |
| 数据库 | 数据持久化 | 待定 |
| AI 网关 | 模型请求转发 | 待定 |

## 设计原则

1. **模块化** — 各组件低耦合高内聚
2. **可扩展** — 支持横向扩容
3. **安全性** — 用户数据加密传输
4. **ARM64 优化** — 充分利用 ARM 架构特性
