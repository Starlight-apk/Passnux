# 项目架构概览

## 整体架构

Passnux 采用前后端分离架构。AI 能力由上游 [new-api](https://gitcode.com/QuantumNous/new-api) 网关提供，Passnux 自身**不直接调用任何 AI 模型**，只负责前端界面、API 代理转发、用户管理与额度分发：

```
┌─────────────┐     ┌──────────────┐     ┌──────────────────┐     ┌─────────────┐
│  前端       │────▶│  Passnux     │────▶│  new-api (上游)  │────▶│  AI 模型    │
│ (Web UI)    │◀────│  API 代理    │◀────│  AI 网关         │     │ (多厂商)    │
└─────────────┘     └──────────────┘     └──────────────────┘     └─────────────┘
                     │    │    │
              ┌──────┘    │    └──────┐
              ▼           ▼           ▼
         ┌────────┐ ┌──────────┐ ┌────────┐
         │ 用户   │ │ 额度     │ │ 对话   │
         │ 管理   │ │ 分发     │ │ 历史   │
         └────────┘ └──────────┘ └────────┘
```

## 核心组件

| 组件 | 职责 | 技术选型 |
| :--- | :--- | :--- |
| 前端 | 用户界面交互（AI 对话、模型切换） | 待定 |
| 后端 | API 代理转发、用户管理、额度分发 | 待定 |
| 数据库 | 用户数据、对话历史、额度记录 | 待定 |
| AI 网关（上游） | 模型请求聚合、负载均衡、密钥管理 | [new-api](https://gitcode.com/QuantumNous/new-api) |

## 设计原则

1. **模块化** — 各组件低耦合高内聚
2. **可扩展** — 支持横向扩容
3. **安全性** — 用户数据加密传输
4. **ARM64 优化** — 充分利用 ARM 架构特性
